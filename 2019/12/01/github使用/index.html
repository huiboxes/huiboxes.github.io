Github是什么？

Github是一个代码托管平台，可以通过此平台找到许多开源项目，可以共享自己的开源项目与他人共同开发、完善，由于只支持Git的版本库格式进行托管，所以名为Github。

Git是什么？工作流程是什么？

Git是一个开源的分布式版本控制系统。

  • Workspace：工作区

  • index/Stage：暂存区

  • Repository：仓库区

  • Remote：远程仓库

    在工作区中修改某些文件 → 对修改后的文件进行快照，然后保存到暂存区 → 提交更新，将保存在暂存区域的文件快照永久转储到Git目录中 → 将本地仓库push到远程仓库

一、准备工作

1.安装Git Bash：在 https://gitforwindows.org/ 下载，然后一直下一步就好。

2.注册Github账户：点这里进入官网,英语不好的建议用chrome，翻译挺好用的。注册时全部默认下一步即可，注册完账号就创建一个新仓库，别的都不管，直接默认的选项下一步即可，等后面没事了一定要有空就打开看看，每个都按一下，摸索一下(不要删除仓库和账号就好)。

二、配置Git Bash：

依次在输入下面的内容，输一行，按一次回车。

Git Bash只需要在工作目录右击鼠标，然后选Git Bash here即可

三、配置Github

1、打开Git Bash → 输入rm -rf ~/.ssh/* 然后回车把现有的ssh key都删掉（千万不能输错，不然就要重装系 统了）→ 输入ssh-keygen -t rsa -b 4096 -c "你的邮箱" → 按三次回车就创建好了，在根目录下的.ssh 目录中有两个文件，id_rsa是私钥，不能给任何人。

​ 紧接着输入cat ~/.ssh/id_rsa.pub → 从ssh_rsa开始复制全部内容。

2.打开https://github.com/settings/keys → 点击New SSH key,title随便输入，在key下的表单内粘贴刚刚复 制的内容，点Add SSH key。

3.回到Git Bash → 输入ssh -T git@github.com如果提示输入yes/no就输入yes，只要没有看到一串包含 successfully的英文，就要重新弄。

  • ​ 如果把key从电脑上删除了，重新生成一个就好了，替换之前的key。
  • ​ 一个电脑只需要一个key。
  • ​ 其他设备如果想的key上传到Github可以和之前的 key共存在Github上
  • ​ 一个key可以访问所有的仓库。
